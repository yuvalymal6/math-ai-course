# DNA של רמה 100 — כיתה י׳

## כלל עיצוב בסיסי — כיתה י
כל המעבדות בכיתה י משתמשות ברקע בהיר בלבד.
Island Protocol (background: #020617) שייך לכיתות יא-יב בלבד.
סגנון מעבדה תקין לכיתה י:
```
border: "1px solid ...", borderRadius: 24,
background: "rgba(255,255,255,0.82)"
```

---

> מסמך זה מגדיר את הסטנדרט המלא לדף תרגיל בפרויקט. נלמד מ-5 קבצים מייצגים:
> `geo-parallelogram`, `algebra-parabola`, `statistics/central`, `geo-rhombus`, `trig-basics`

## 1. מבנה כללי — הדפוס החוזר

כל דף תרגיל ברמה 100 מכיל את **כל** הרכיבים הבאים:

### 1.1 שלוש רמות קושי
| רמה | תווית | צבע | קומפוננטה |
|------|--------|------|-----------|
| מתחיל | basic | ירוק `#16A34A` | `LadderBase` — העתקת פרומפטים + דיווח עצמי |
| בינוני | medium | כתום `#EA580C` | `LadderMedium` — כתיבת פרומפט + ולידציה (ניקוד 75+) |
| מתקדם | advanced | אדום `#DC2626` | `LadderAdvanced` — `MasterPromptGate` + ניקוד 90+ |

### 1.2 רכיבים חובה לכל רמה
- **דיאגרמת SVG שקטה** — ללא מספרים, ללא תשובות. רק צורות, תוויות קודקודים, צבעי תפקיד
- **טקסט השאלה** — מנוסח בעברית, מחולק לסעיפים (א׳/ב׳/ג׳/ד׳)
- **מוקשים נפוצים (Pitfalls)** — 2-3 כרטיסי אזהרה עם כותרת + הסבר
- **פרומפט זהב (Golden Prompt)** — פרומפט ראשי מלא להעתקה (חוץ מרמה מתקדמת)
- **סולם פרומפטים (Ladder)** — שלבים עם פרומפטים לכל סעיף
- **מעבדה אינטראקטיבית (Lab)** — סליידרים + SVG דינמי + אריחי נתונים

### 1.3 רכיבי עטיפה
- **`SubtopicProgress`** — סרגל התקדמות כללי
- **`MarkComplete`** — כפתור סיום
- **`LabMessage`** — הודעות מותנות במעבדה
- **כפתור חזרה** — לינק לדף האב

## 2. הפרומפט הזהב — מבנה אחיד

כל פרומפט זהב ברמה 100 עוקב אחרי פרוטוקול 3 חלקים:

```
1. הגדרת תפקיד: "אתה המורה הפרטי שלי / אני תלמיד כיתה י׳"
2. מגבלה: "אל תפתור עבורי — שאל אותי שאלות מכווינות"
3. שיטת עבודה: "סרוק את הנתונים בלבד. תעצור אחרי כל שלב ותחכה שאגיד להמשיך"
```

**דוגמה (geo-parallelogram, basic):**
```
היי, אני תלמיד/ה כיתה י' ומצרף/ת שאלה על מקבילית. אני רוצה שתהיה המורה הפרטי שלי —
תסביר לי את הלוגיקה ולא תיתן תשובות ישירות.

אל תפתור עבורי — שאל אותי שאלות מכווינות.
סרוק את הנתונים בלבד.
אל תמהר, תסביר לי על כל שלב. בסיום הסריקה של הנתונים שהדבקתי,
תגיב אך ורק: ״אני מוכן להמשיך.״
```

**דוגמה (trig-basics, medium):**
```
אתה מתפקד כרגע כמורה המקצועי שלי לטריגונומטריה. אני זקוק להדרכה המלאה שלך.
דבר ראשון, סרוק נתונים ועצור כדי להראות לי שהבנת את התרגיל.
אני מצפה ממך לא לגלות את הפתרון בשום שלב, אלא להוביל אותי לבניית הפתרון הנכון בעצמי.
תסביר לי את הלוגיקה שלב אחר שלב ותעצור בסיום כל סעיף ותחכה לשאלת המשך.
```

**אורך אופייני:** 150-400 תווים. פרומפטי שלבים: 50-120 תווים (משפט אחד ממוקד).

## 3. פרומפטי שלבים — סגנון הניסוח

### ציווי ישיר בגוף שני
- "תדריך אותי איך..." / "תסביר לי..." / "הנחה אותי..."
- "תוכל להנחות אותי איך נבנה משוואה מתאימה?"

### מגבלות מפורשות
- "אל תפתור" / "אל תיתן תשובה סופית"
- "תעצור אחרי כל הסבר קצר"

### הפניה לנתונים
- "כיתה א': 78,79,80,81,82. כיתה ב': 60,70,80,90,100."
- "במשולש ישר זווית ABC, ∠B = 90°..."

## 4. דיאגרמות SVG שקטות — כללים

| כלל | פירוט |
|------|--------|
| אין מספרים | לא `6`, לא `82`, לא `√52` |
| אין תשובות | לא `שטח = 48`, לא `x = 7` |
| תוויות קודקודים בלבד | A, B, C, D, O, E, F |
| צבעי תפקיד | ענבר `#f59e0b` (בסיסי), סגול `#a78bfa` (מתקדם), אמרלד `#34d399` (פתרון), צפחה `#64748b` (עזר) |
| קווים מוסתרים | `strokeDasharray="4,3"`, צבע `#334155` |
| גודל | `className="w-full max-w-sm mx-auto"`, `viewBox` בלבד |

**דוגמה (geo-rhombus, basic):** מעוין ABCD עם אלכסונים מקווקווים (ענבר + סגול), נקודת חיתוך O (ציאן), סימוני שוויון על 4 צלעות, קשת זווית ב-A. **אפס מספרים.**

## 5. מעבדות אינטראקטיביות — דפוס

### מבנה אחיד
1. **סליידרים** — `<input type="range">` לכל פרמטר
2. **חישוב חי** — ללא כפתור submit
3. **SVG דינמי** — משתנה בכל תזוזת סליידר
4. **אריחי נתונים** — `grid grid-cols-N gap-3`, כל אריח: תווית + ערך
5. **תווית תחתונה** — פרשנות מתמטית ("הריבוע הוא מקרה פרטי של מעוין!")

### דוגמאות מרמה 100
| קובץ | מעבדה | סליידרים | אריחים |
|-------|--------|----------|--------|
| geo-parallelogram | `LabBasic` | 1 (זווית 30-150) | 4 זוויות + מלבן! |
| geo-rhombus | `RhombusLab` | 1 (זווית 30-150) | 4 זוויות + אלכסון 90° |
| trig-basics | `LadderLab` | 2 (x, זווית A) | 6: AB, AC, שטח, זווית C |
| algebra-parabola | `ParabolaLab` | 3 (a, b, c) | 3: קודקוד, כיוון, חיתוך Y |
| statistics/central | `BarChartLab` | 10 (ציונים) | 3: ממוצע, חציון, שכיח |

### מה צריך לשנות בכל קובץ

#### `analytic/basics` (הכי קרוב לרמה 100)
- **Labs:** וודא סגנון בהיר (border 1px, background rgba לבן) — ללא Island Protocol
- **הכל השאר:** מלא ותקין

#### `analytic/line`
- **SVGs:** הסר טקסטי משוואות מהדיאגרמות הסטטיות (השאר רק תוויות קודקודים)
- **Labs:** וודא סגנון בהיר — ללא Island Protocol

#### `analytic/problems`
- **SVGs:** הסר קואורדינטות מספריות מקודקודים בדיאגרמות סטטיות
- **Labs:** וודא סגנון בהיר — ללא Island Protocol

#### `trig-applications`
- **Labs:** וודא סגנון בהיר — ללא Island Protocol
- **הכל השאר:** תקין — SVGs שקטים, מבנה מלא

#### `statistics/center`
- **ולידציה:** החלף `checkPrompt()` מותאם ב-`calculatePromptScore` + `contextWords`
- **Lab:** וודא סגנון בהיר. שקול הוספת מעבדות ייחודיות לרמות medium/advanced
- **SubtopicProgress:** מיובא אך לא מוצג — הוסף

#### `statistics/dispersion`
- **סגנון:** וודא סגנון בהיר (כמו כל שאר כיתה י׳) — ללא Island Protocol
